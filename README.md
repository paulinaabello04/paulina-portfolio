# Portfolio Creativo — Paulina Abelló

Proyecto final desarrollado para la practica **Web Foundations (UDIT)**.  
Este trabajo consiste en la creación de una **plantilla profesional de portfolio web**, desarrollada con tecnologías web estándar y enfocada en demostrar el dominio de los fundamentos vistos durante el curso.


## Descripción del proyecto

El proyecto es un **portfolio personal de diseño gráfico y digital** donde presento una selección de trabajos, información personal, habilidades y contacto.

El objetivo principal ha sido construir una web clara, funcional y visualmente coherente, con un enfoque editorial y minimalista, priorizando:

- La estructura semántica
- La legibilidad del código
- El diseño responsive
- La manipulación directa del DOM sin frameworks


## Tecnologías utilizadas

### HTML5
- Estructura semántica correcta
- Uso de secciones (`header`, `main`, `section`, `footer`)
- Atributos de accesibilidad (ARIA)
- Separación clara entre contenido y presentación

### CSS3 (Vanilla)
- Uso extensivo de **custom properties (variables CSS)** para colores, tipografía y espaciados
- Layout construido con **Flexbox y CSS Grid**
- Diseño **responsive** para desktop, tablet y móvil
- Implementación de **modo claro / oscuro** mediante `data-theme`
- Organización del CSS por bloques (base, layout, componentes, responsive)

### JavaScript (ES6+)
- JavaScript moderno (`const`, `let`, funciones flecha)
- Manipulación del DOM sin librerías externas
- Cambio de idioma (ES / EN)
- Cambio de tema (light / dark)
- Sistema de filtros de proyectos
- Modal para detalle de proyectos
- Galería de imágenes y lightbox
- Gestión de eventos de scroll e interacción

No se ha utilizado ningún framework CSS externo (Bootstrap, Tailwind, etc.), cumpliendo estrictamente con las indicaciones del briefing del curso.


## Funcionalidades principales

- Cambio de idioma Español / Inglés
- Toggle de tema claro / oscuro
- Filtros de proyectos por categoría
- Modal con información detallada de cada proyecto
- Galería de imágenes con lightbox
- Diseño responsive
- Barra de progreso de scroll
- Botón “Back to top”


## Enfoque técnico y arquitectura

- El JavaScript está organizado en funciones con responsabilidades claras, evitando lógica innecesariamente compleja.
- El estado de la aplicación (idioma, tema, modal activo) se gestiona de forma explícita.
- El CSS se apoya en variables para mantener coherencia visual y facilitar ajustes globales.
- Se ha aplicado una mentalidad de **mejora progresiva**, garantizando que el contenido principal sea accesible incluso si JavaScript falla.


## Uso de Inteligencia Artificial

El uso de Inteligencia Artificial ha estado documentado, siguiendo las indicaciones del curso.

La IA se ha utilizado como herramienta de apoyo. Principalmente para:

- Planificar funcionalidades antes de implementarlas
- Resolver errores concretos durante el desarrollo
- Revisar fragmentos de código y proponer mejoras
- Ayudar a entender conceptos de JavaScript y CSS

El flujo de trabajo ha sido el siguiente:

1. Definición del problema o funcionalidad
2. Consulta y análisis de posibles soluciones
3. Implementación manual del código
4. Pruebas, depuración y ajustes propios
